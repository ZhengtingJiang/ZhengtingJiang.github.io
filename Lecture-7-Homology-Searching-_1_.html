Lecture 7 Homology Searching (1)
Here we will discuss about the searches for similarities. The main question is: what is the function
of the new gene? If we want a lazy investigation without doing experiments, we could find a set of
similar proteins, and identify similarities and differences, for long proteins, it is often good to
identify domains first and then compare the corresponding subsequences separately. Here, a
domain is a unit of function, multi-domain proteins have a compound function.
As we mentioned before, there is a term called homology, it means sharing a common ancestor,
which is a binary property, answers are yes or no. Common ancestry makes it more likely that
genes share the same function, so it is a nice tool, because when a known gene G is homologous
to an unknown gene X, we gain a lot of information on X by transferring what we know about G.
A question is can we just transfer information about structure and/or function? Structure and
function are more conserved than sequence, and the function is made by structure is made by
sequence. So, if the sequences already tell us it is the same thing (homolog), then certainly the
structures and functions are supposed to be the same, this works most of the time, but there are
some cases where likely homology does not bear out.
If we want to find sequences homolog, there is a problem, remember what we introduced before?
Mind the gap! There are far more sequence data than structural and functional data, the
exponential growth of databases makes it difficult to find sequences homolog in a fast and
accuracy way. We need to fill this gap by analysis and prediction pipelines.
7.1 Sequence searching - definition
The task is:
Query: short, new sequences (around 1000 residues)
Databased (searching space): very many sequences
Goal: find sequences related to the query
We want:
A fast tool
Primarily a filter: most sequences will be unrelated to the query
Fine-tune the alignment later
So a heuristic alignment will be useful. The method performs fast approximation of dynamic
programming, such as BLAST and FASTA. The dynamic programming algorithm has complexity
which is too slow for large databases with high query traffic. Let us consider the task of
searching WISS-PROT against a query sequence. Say, our query sequence is 362 amino acids long,
SWISS-PORT release 38 contains 29085265 amino acids. If we want to find local alignments via
dynamic programming, it would entail matrix operations, many servers handle
thousands of such queries a day, and using the DP algorithm for this is clearly prohibitive!
Therefore, today, the methods BLAST and PSI-BLAST are discussed to show a few of the tricks
people have come up with to make alignment and database searching fast, while not losing too
much quality. The earlier method FASTA is also discussed as a reference.
7.2 Basic Local Alignment Searching Tool (BLAST)
The basic idea is to discard putatively unrelated sequences fast, and high scoring segments have
well conserved (almost identical) part, as well-conserved segments are identified, extend these to
the real alignment. However, it is only a heuristic! It can only help in solving a certain class of
problems, but make no guarantees.
Here, what does well-conserved mean in BLAST? We should introduce the principles of BLAST.
BLAST works with -words, here is a parameter, which is different for DNA (>10) and proteins (2

The BLAST algorithm has 3 basic steps.
1. Process the query sequence: extract all the -words.
2. Scan for -similar matches in database (fast step).
3. Extend these to alignments (slow step).

Step 2. Scanning the database with DFA (deterministic finite-state automaton).
Here, it searches database for all occurrences of query words, which can be a massive task. The
approach is to build a DFA that recognizes all query words, then run DB sequences through DFA,

Back to our topic, it uses all the -similar -words to build the Finite State Machine, scan for exact
matches.
Step 3. Non-gapped extension (old BLAST).
Having the list of matches (hits) we extend alignment in both directions, till the sum of scores
drops below some level from the best known. This extends hits in both directions without
allowing gaps, and terminates extension in one direction when score falls certain distance below
best score for shorter extension. It returns segment pairs scoring at least a threshold score .

The first thing is two-hit method, which is a justification of the old BLAST. Old extension step
typically accounts for 90% of BLAST's execution time. Its key idea is to do extension only when
there are two non-overlapping hits on the same diagonal within distance of each other (
amino acids). To maintain sensitivity, lower parameter is used, because it could help get more

Next, we introduce the PSI-BLAST. It is Position-Specific Iterated BLAST. A profile (called PSSM by
BLAST) is derived from the result of the first searching using a single query sequence. Database is
searched against the profile instead of a sequence in subsequence rounds. Up to 3-10 iterations
are recommended.
The steps of PSI-BLAST:
1. Query sequences are first scanned for the presence of so-called low-complexity regions, i.e.
regions with a biased composition likely to lead to spurious hits are excluded from alignment.
2. The program then initially operates on a single query sequence by performing a gapped
BLAST search.
3. The program takes significant local alignments hits found, constructs a multiple alignment
and abstracts a PSSM from this alignment.
4. The database is rescanned in a subsequent round, now using the PSSM, to find more
homologous sequences. Iteration continues until user decides to stop or the search has
converged.
We have already talked about the definition of profile. The low-complexity sequence has not
talked about yet. For example, AAAAA... or AYLAYLAYLA... or AYLLYAALY ... Low-complexity
(sub)sequences have abiased composition and contain less information than high-complexity
sequences. Because of the low information content, they often lead to spurious hits without a
biological basis (for example, you cannot tell whether a poly-A sequence [AAAAA...] is more similar
to a globin, an immunoglobulin or a kinase sequence). That is why BLAST filters low-complexity
regions in the query sequence out.

7.4 FASTA
We will introduce a piece of history :FASTA. It is the first homology searching method and until
gapped-BLAST and PSI-BLAST appeared, FASTA has in fact been the better method - unlike the
impression created by the BLAST folks. It compares a given query sequence with a library of
sequences and calculates for each pair the highest scoring local alignment. Similar to BLAST,
speed is obtained by delaying application of the dynamic programming technique to the moment
where the most similar segments are already identified by faster and less sensitive techniques.
The first three steps are quick but the last step for full dynamic programming alignment over the
final region will be slow! . We could introduce more for step 1. Rapid identical word
searches: Search for -tuples of a certain size within a specified bandwidth along search matrix
diagonals. For not-too-distance sequences, little sensitivity is lost while speed is greatly increase.
Technique emplyed is known as hash coding or hashing: a lookup table is constructed for all
words in the query sequence, which is then used to compare all encountered words in each
database sequence. Sequence positions in query are hashed!
Query: ERLFERLACER.....
DB: MERIFERLACACTR....
We only need to go through the DB sequence once: for each word encountered (ER, RL, LF, ...),
check the query hash list for the word. If found, you immediately have the query sequence
positions of the word, you also know the position you are at in the DB sequence, and so you can
fill in the matrix with diagonals right away! Algorithmic speed therefore is linear with DB
sequence length or . Compare this to finding all word match positions without a hash list
.
